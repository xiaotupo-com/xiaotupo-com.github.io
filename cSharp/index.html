C# （发音：See Sharp）是一种面向对象的，面向组件的编程语言，运行在微软 .NET 平台，是一种安全的语言。

所有 C# 类型（包括 int 和 double 等基元类型）均继承自一个根 object 类型。

# .NET 体系结构

TIP

C# 程序在 .NET 上运行，而 .NET 是名为公共语言运行时 (CLR) 的虚执行系统和一组类库。 CLR 是 Microsoft 对公共语言基础结构 (CLI) 国际标准的实现。 CLI 是创建执行和开发环境的基础，语言和库可以在其中无缝地协同工作。

用 C# 编写的源代码被编译成符合 CLI 规范的中间语言 (IL)。 IL 代码和资源（如位图和字符串）存储在扩展名通常为 .dll 的程序集中。 程序集包含一个介绍程序集的类型、版本和区域性的清单。

执行 C# 程序时，程序集将加载到 CLR。 CLR 会直接执行实时 (JIT) 编译，将 IL 代码转换成本机指令。 CLR 可提供其他与自动垃圾回收、异常处理和资源管理相关的服务。 CLR 执行的代码有时称为“托管代码”。而“非托管代码”被编译成面向特定平台的本机语言。

语言互操作性是 .NET 的一项重要功能。 C# 编译器生成的 IL 代码符合公共类型规范 (CTS)。 通过 C# 生成的 IL 代码可以与通过 .NET 版本的 F#、Visual Basic、C++ 生成的代码进行交互。 还有 20 多种与 CTS 兼容的语言。 单个程序集可包含多个用不同 .NET 语言编写的模块。 这些类型可以相互引用，就像它们是用同一种语言编写的一样。

除了运行时服务之外，.NET 还包含大量库。 这些库支持多种不同的工作负载。 它们已整理到命名空间中，这些命名空间提供各种实用功能。 这些功能包括文件输入输出、字符串控制、XML 分析、Web 应用程序框架和 Windows 窗体控件。 典型的 C# 应用程序广泛使用 .NET 类库来处理常见的“管道”零碎工作。

# 类型和变量

C# 的值类型进一步分为：简单类型、枚举类型、结构类型、可以为 null 的值类型和元组值类型。 C# 引用类型又细分为类类型、接口类型、数组类型和委托类型。

# 顶级语句

使用顶级语句的先决条件：

需要将计算机设置为运行 .NET 6，其中包括 C# 10 编译器。 自 Visual Studio 2022 或 .NET 6 SDK 起，开始提供 C# 10 编译器。

没有使用顶级语句时：

using System;

namespace Application
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello World!");
        }
    }
}

使用顶级语句：

// See https://aka.ms/new-console-template for more information
Console.WriteLine("Hello, World!");

顶级语句必须在类型定义和命名空间之前。

# 嵌套类

定义在类、构造或接口中的类型称为嵌套类型。下面例子中使用了 C# 最新的顶级语句，Hello 就是一个和 main() 函数同级的内部类：

int count = 100;

void printCount()
{
    Console.WriteLine(count);
}

// 调用下面两个函数
Hello.Print();
printCount();

class Hello
{
    static void PrintHello()
    {
        Console.WriteLine("Hello, World!");
    }

    public static void Print()
    {
        PrintHello();
        //printCount(); 无法访问类外的类型
    }
}

不管外部类是什么类型，嵌套类（内部类）默认访问权限均为 private；仅可在其内部访问。

我们可以修改 PrintHello() 函数的访问权限为 public，这样就可以在外部类中访问了。

类的嵌套类型访问符可以是：protected、protected、 internal、protected internal、private 或 private protected。

另请注意，使嵌套类型在外部可见违反了代码质量规则 CA1034“嵌套类型不应是可见的”。

构造的嵌套类型可以是 public、internal 或 private。

# ref 引用

C# 中的 ref 引用和 c 语言中的指针类型，可以间接访问其他变量。

下面例子中创建了一个引用 a 并且指定引用自 b，这样一来不论修改 a 还是 b 两个值都会变化，其实 a 相当于一个指针。

int b = 10;
int c = 20;

// 定义引用类型 a ，并且指定引用 b 的地址
ref int a = ref b;

a = 100;

Console.WriteLine(b); // output: 100

b = 200;
Console.WriteLine(a); // output: 200

// 修改 a 的引用为 c
a = ref c;
Console.WriteLine(a); // output: 20

# ref 和 readonly

readonly 修饰符可以应用于 ref 局部变量和 ref 字段。 readonly 修饰符会影响其右边的表达式。

ref readonly int a； 可以修改 a 的引用，不可修改 a 引用的值

int b = 10;
int c = 20;

// ref readonly int a； 可以修改 a 的引用，不可修改 a 引用的值
ref readonly int a = ref b;
Console.WriteLine(a); // 可以读取，不可修改 a
a = ref c; // 可以修改 a 的引用
Console.WriteLine(a); // output: 20

# 创建类

C# 中的构造函数有多种写法，下面演示了几种不同的构造函数，注释起来的都是合法的。

namespace helloCSharp
{
    internal class Point
    {
        //public Point(double x, double y)
        //{
        //    X = x;
        //    Y = y;
        //}

        //public Point(double x, double y) => (X, Y) = (x, y);

        public Point(double x) => this.X = x;

        //public Point(double x, double y)
        //{
        //    this.X = x;
        //    this.Y = y;
        //}

        public Point(double x = 10, double y = 10)
        {
            this.X = x;
            this.Y = y;
        }

        public double X { get; set; }
        public double Y { get; set; }

        public override string ToString()
        {
            return $"x: {X}\ny: {Y}";
        }
    }
}

var point1 = new Point();

Console.WriteLine(point1.ToString());
