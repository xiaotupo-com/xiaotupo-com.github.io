Rust 中的变量和其他语言基本一致，但是也有一些不同的地方，现在先来定义一些基本的变量。

fn main() {
    // rust 中的变量使用 let 关键字来定义
    let size: i32; // 定义一个 i32 类型的变量，默认变量是不可变的，在这里我们可以不进行初始化，但是我们只能初始化一次不可变变量
    size = 100; // 这里初始化了不可变变量 size，之后将不能再次修改
    println!("size: {}", size);

}

该例子中定义了一个 i32 类型的变量，该变量是不可变的，就是定义后不可修改，所以在定义是就要进行初始化，否则编译器将会提示错误。Rust 中要定义可变的变量，需要使用关键字 mut。

# 定义可变变量

fn main() {
    // rust 中的变量使用 let 关键字来定义
    let mut size: i32; // 使用 mut 关键字，定义一个 i32 类型的可变变量

    size = 200;

    println!("size: {}", size);

    size = 1000;

    println!("size: {}", size); // 如果把改行注释起来，编译器将会在 size = 1000; 这行提示变量未使用，因为修改了变量的值之后未使用所以有提示
    // 可以看到 Rust 的编译器还是很智能的
}

# 常量

类似于不可变变量，常量 (constants) 是绑定到一个名称的不允许改变的值，不过常量与变量还是有一些区别。

常量与不可变变量的区别有：

  • 不能对常量使用 mut 关键字
  • 定义常量是必须指定常量类型
  • 常量使用 const 关键字定义
  • 常量只能被设置为常量表达式，不能是其他在运行时计算出来的值
fn main() {
    // 定义一个常量类型
    const WIDTH: f64 = 45.34562;
}

# 隐藏

在 Rust 中我们可以多次定义同名的变量，后一次可以隐藏（Shadowing）前一次变量。

fn main() {
    // rust 中的变量使用 let 关键字来定义
    let size: i32; // 定义一个 i32 类型的可变变量

    size = 200;

    println!("size: {}", size);

    let size = size + 1000; // 再次定义名为 size 的变量，并且使用前一次 size 的值

    println!("size: {}", size); // 如果把改行注释起来，编译器将会在 size = 1000; 这行提示变量未使用，因为修改了变量的值之后未使用所以有提示

}

隐藏可以定义不同类型的变量，如下例子：

let spaces = "   ";
let spaces = spaces.len();

# Rust 中的基础类型

Rust 中的数据类型分为：标量（scalar）和复合（compound）两种。

标量类型分为：整型、浮点型、布尔、字符 等几种。

Rust 有两个原生的复合类型：元组（tuple）和数组（array）。

# 整型溢出open in new window

比方说有一个 u8 ，它可以存放从零到 255 的值。那么当你将其修改为 256 时会发生什么呢？这被称为 “整型溢出”（“integer overflow” ），这会导致以下两种行为之一的发生。当在 debug 模式编译时，Rust 检查这类问题并使程序 panic，这个术语被 Rust 用来表明程序因错误而退出。第九章 “panic! 与不可恢复的错误”open in new window 部分会详细介绍 panic。

使用 --release flag 在 release 模式中构建时，Rust 不会检测会导致 panic 的整型溢出。相反发生整型溢出时，Rust 会进行一种被称为二进制补码 wrapping（two’s complement wrapping）的操作。简而言之，比此类型能容纳最大值还大的值会回绕到最小值，值 256 变成 0，值 257 变成 1，依此类推。程序不会 panic，不过变量可能也不会是你所期望的值。依赖整型溢出 wrapping 的行为被认为是一种错误。

为了显式地处理溢出的可能性，可以使用这几类标准库提供的原始数字类型方法：

  • 所有模式下都可以使用 wrapping_* 方法进行 wrapping，如 wrapping_add
  • 如果 checked_* 方法出现溢出，则返回 None值
  • 用 overflowing_* 方法返回值和一个布尔值，表示是否出现溢出
  • 用 saturating_* 方法在值的最小值或最大值处进行饱和处理

# 浮点型

Rust 中分为两种浮点型分别为 f32 和 f64。默认类型是 f64。

fn main() {
    let x = 2.0; // f64

    let y: f32 = 3.0; // f32
}

浮点数采用 IEEE-754 标准表示。f32 是单精度浮点数，f64 是双精度浮点数。

# 布尔

Rust 的布尔值有两个 true 和 false。使用 bool 关键字代表布尔型。

fn main() {
    let t = true;

    let f: bool = false; // with explicit type annotation
}

# 字符类型

Rust 中的 char 类型用来表示当个字符，和 c 语言中的 char 类型，但是 Rust 中的 char 类型的大小为 4 个字节。

fn main() {
    let c = 'z';
    let z: char = 'ℤ'; // with explicit type annotation
    let heart_eyed_cat = '😻';
}

# 元组类型

元组类型是一个将多个不同类型的值组织起来的一种类型，元组的长度固定，一旦声明，其长度将不可改变。

元组的类型使用圆括号括起来，用逗号分隔多个值。

fn main() {
    let tup: (i32, f32, bool) = (1, 100.43, true); // 创建了一个不可变元组

    let (a, b, c) = tup; // 结构元组，创建三个变量 a,b,c，并且从 tup 中按位置一一对应解构出来

}

访问元组内的值用 . 跟索引即可，索引值从 0 开始。

有一种不带任何值的元组，这种元组的名字叫做**单元（unit）**元组。这种值以及对应的类型都写作 ()，表示空值或空的返回类型。如果表达式不返回任何其他值，则会隐式返回单元值。

# 数组类型

Rust 中的数组类型和其他语言类似，一个数组中的每个值的类型要相同。

fn main() {
    let a = [1, 2, 3, 4, 5];
    let a = [3; 5]; // 定义一个包含5个3的数组。
}
