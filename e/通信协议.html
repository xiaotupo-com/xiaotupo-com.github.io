SPI、I2C 和 UART 比 USB、以太网、蓝牙和 WiFi 等协议慢很多，但它们简单得多，硬件和系统资源使用更少。SPI、I2C 和 UART 非常适合微控制器之间以及微控制器和传感器之间的通信，无需传输大量高速数据。

# 串行与并行通信

电子设备通过设备之间物理连接的电线发送数据位来相互通信。这有点像一个单词中的字母，只存在于单词表中的 26 个字母内，一个位是二进制的，只能是 1 或 0。通过电压的快速变化，位从一个设备转移到另一个设备。在一个操作在 5V 的系统中，0 位作为 0V 的短脉冲进行通信，1位通过 5V 的短脉冲进行通信。

数据位可以并行或串行形式传输。在并行通信中，数据位均同时发送，每个数据位均通过单独的导线发送。下图显示二进制字母"C"的并行传输（01000011）：

# SPI 协议

SPI 是串行外设接口（Serial Peripheral Interface）的缩写，是一种高速的，全双工，同步的通信总线，并且在芯片的管脚上只占用四根线，节约了芯片的管脚，同时为 PCB 的布局上节省空间，提供方便，正是出于这种简单易用的特性，越来越多的芯片集成了这种通信协议。

SPI 的一个独特好处是数据可以不间断地传输。任何数量的位都可以在连续流中发送或接收。使用 I2C 和 UART，数据以数据包方式发送，仅限于特定数量的位。启动和停止条件定义每个数据包的开始和结束，因此数据在传输过程中中断。

通过 SPI 通信的设备处于主从关系中。主机是控制装置（通常是微控制器），而从机（通常是传感器、显示器或内存芯片）接受主机的指令。SPI 最简单的配置是单主机、单从机系统，但一个主机可以控制多个从机。

  • MOSI（主机输出/从机输入）–主机向从机发送数据的线
  • MISO（主机输入/从机输出）–从机向主机发送数据的线
  • SCLK（时钟） -时钟信号线
  • SS/CS（从机选择/芯片选择）-用于主机选择给哪个从机发送数据的线

# SPI 的工作原理

  • 时钟

时钟信号用于同步主机和从机之间的数据采样。每个时钟周期传输一位数据，因此数据传输的速度由时钟信号的频率决定。SPI 通信始终由主发起，并由主机配置和产生时钟信号。

设备共享时钟信号的任何通信协议都称为同步通信协议。SPI 是一种同步通信协议。也有不使用时钟信号的异步方法。例如，在 UART 通信中，双方被设置为预先配置的 baud 速率，该速率和时间决定数据传输的速度和时间。

SPI 中的时钟信号可以使用时钟极性和时钟相位的特性进行修改。这两个属性协同工作，以定义位的输出时间和采样时间。时钟极性可以由主机设置，以便对时钟周期的上升沿或下降沿进行位的输出和采样。时钟相位可以设置为在时钟周期的第一边缘或第二边缘进行输出和采样，无论它是上升还是下降。

  • 从机选择

主机可以通过将从机的 CS/SS 线设置为低电平来选择它想与哪个从机通信。在闲置、非传输状态下，从机选择线保持在高压水平。主机上如果可以提供多个 CS/SS 引脚，允许多个从机并行连接。如果只有一个 CS/SS 引脚存在，多个从机可以通过菊花链的形式连接到主机。

  • MOSI 和 MISO

主机通过串行 MOSI 线，逐位的发送数据给从机。从机从主机的 MOSI 管脚接收数据。主机给从机发送数据，通常首先发送最高有效位。

从机也可以通过 MISO 串行线给主机回发数据。数据由从机回发给主机通常首先发送最低有效位。

# SPI 数据传输步骤

  • 主机输出时钟信号

  • 主机拉低 SS/CS 片选线，激活对应从机

  • 主机通过 MOSI 线发送数据给从机，一次发送一位，从机读取收到的数据位

  • 如果需要响应，从机则通过 MISO 线返回数据，一次发送一位，主机读取收到的数据位

# SPI 的优缺点

  • 优点
    • 没有启动和停止位，因此数据可以连续流式传输而不中断；
    • 没有像 I2C 这样复杂的从机寻址系统；
    • 数据传输率高于 I2C（几乎是 I2C 的两倍）；
    • 分离 MISO 和 MOSI 线，以便可以同时发送和接收数据。
  • 缺点
    • 使用四根电线（I2C 和 UART 使用两根线）；
    • 不确认数据已成功接收（I2C 有此）；
    • 没有任何形式的错误检查，如在 UART 中的校验位；
    • 只允许一个主机

# 时钟极性&时钟相位

SPI的极性Polarity和相位Phase，最常见的写法是CPOL和CPHA。

Edge 边沿，即时钟发生变化的时刻，上边沿（rising edge）和上边沿（falling edge）。

前一个边沿（第一个边沿），对于开始电压为高电平1，那么就是1变成0的时候，对于开始电压为低电平0，那么就是0变成1的时候。

后一个边沿（第二个边沿），对于开始电压为高电平1，那么就是0变成1的时候（极在第一次1变成0之后，才可能有后面的0变成1），对于开始电压为低电平0，那么就是1变成0的时候。

  • 极性=CPOL (极性就是CLK空闲时的状态，可能为高电平或低电平)
  • 相位=CPHA (相位代表在时钟的第几个边沿采集数据，0对应第一个边沿，1对应第二个边沿)

CPOL 和 CPHA 的4中组合：

模式组合值
1CPOL=0,CPHA=0
2CPOL=0,CPHA=1
3CPOL=1,CPHA=0
4CPOL=1,CPHA=1

  • CPHA=0，表示第一个边沿采集数据

    • 对于 CPOL=0，IDLE时为低电平，第一个边沿是从低变到高，所以是上升沿
    • 对于 CPOL=1, IDLE时为高电平，第一个边沿是从高变到低，所以是下降沿

  • CPHA=1，代表第二个边沿采集数据

    • 对于 CPOL=0，IDLE时为低电平，第二个边沿是从高变到低，所以是下降沿
    • 对于 CPOL=1, IDLE时为高电平，第二个边沿是从低变到高，所以是上升沿
